# Thinking Machines Angle

## The Pitch

This project is strong PR for Thinking Machines because it makes a clean public argument:

`Custom-model infrastructure is not just for enterprise copilots. It can unlock frontier-beating performance for languages and communities that giant general models still underserve.`

## Why This Story Works

- It uses Tinker for a real fine-tuning workflow, not just a sponsor logo slide
- It produces a live public-facing demo
- It has benchmark evidence rather than anecdotal screenshots
- It targets a socially meaningful but technically difficult problem
- It shows that customization can beat scale on a narrow task

## Talking Points To Use With Sponsor Staff

- We chose a 30B-total / 3B-active MoE because it makes the efficiency story legible.
- Tinker made it practical to adapt the model quickly enough to ship during a hackathon workflow.
- The repo already contains the data, training, eval, and deployment stack, so additional credits produce visible output rather than speculative R&D.
- This is exactly the kind of project that makes "custom models" feel concrete to outsiders.

## Clean Sponsor-Friendly Line

`Thinking Machines helped make a customized Tuvaluan model possible, and the result is a public proof that fine-tuning infrastructure can preserve and strengthen languages that the default AI stack still misses.`

## Ask

If you talk to the Thinking Machines crew directly, ask for credits in terms of outputs:

- larger overlap benchmarks against frontier models
- another tuning round using live feedback signals
- broader task coverage beyond translation
- a polished public case study with numbers
